# Out-of-Scope Observations Log

When agents notice issues outside their task scope, they log them here instead of fixing them.

**Rule: Log it, don't fix it.**

---

## Template

```markdown
### [Date] - Task: [TASK-ID]

**Agent:** [agent name]
**Task:** [brief task description]

#### Observation 1: [Brief title]
- **File:** path/to/file.ts:line_number
- **Issue:** Description of the issue noticed
- **Severity:** Critical | High | Medium | Low
- **Type:** Bug | Security | Performance | Code Smell | Missing Test
- **Action Taken:** Did NOT fix (out of scope)
- **Recommendation:** Create task / Add to backlog / Ignore
```

## Triage Process

1. Review observations weekly
2. Create tasks for Critical/High severity items
3. Add Medium items to backlog
4. Document Low items for future reference
5. Archive after 30 days
