- Frontend component creation needed
- Design system generation requested
- `/devteam:design` command used

## Design System Generation

When generating a design system:

```javascript
const designSystem = {
  colors: {
    palette: selectPalette(requirements),
    semantic: {
      primary: palette.primary,
      secondary: palette.secondary,
      accent: palette.accent,
      success: '#22C55E',
      warning: '#F59E0B',
      error: '#EF4444',
      info: '#3B82F6',
    },
  },
  typography: {
    fontFamily: {
      heading: selectHeadingFont(style),
      body: selectBodyFont(style),
      mono: 'JetBrains Mono',
    },
    scale: [12, 14, 16, 18, 20, 24, 30, 36, 48, 60, 72],
  },
  spacing: [0, 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96],
  borderRadius: {
    none: '0',
    sm: '0.125rem',
    base: '0.25rem',
    md: '0.375rem',
    lg: '0.5rem',
    xl: '0.75rem',
    '2xl': '1rem',
    full: '9999px',
  },
  shadows: generateShadowsForStyle(style),
  animations: generateAnimationsForStyle(style),
};
```

## Component Variants

For each style, generate appropriate component variants:

- Buttons (primary, secondary, ghost, outline)
- Inputs (default, focused, error, disabled)
- Cards (elevated, outlined, flat)
- Navigation (top, side, mobile)
- Modals (centered, fullscreen, drawer)
- Tables (striped, bordered, minimal)
- Forms (inline, stacked, floating labels)

## Accessibility

All designs include:
- WCAG 2.1 AA contrast ratios
- Focus indicators
- Screen reader support
- Keyboard navigation
- Reduced motion variants

1. Design tokens (JSON/CSS variables)

3. Figma-compatible specifications
4. Tailwind config
5. Documentation
